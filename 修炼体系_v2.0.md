## 四维战力

每个修炼者有四维属性，从修炼事件日志动态计算：

| 维度 | 含义 | 主要来源 |
|------|------|---------|
| **coding** | 编码能力 | 完成任务、通过考核 |
| **project** | 工程能力 | 完成任务、项目实践 |
| **theory** | 理论深度 | 考核答题、阅读资源、写文章 |
| **tools** | 工具掌握 | 解锁新工具、配置文件 |

- 每段有**基础值**（随段位指数增长）
- 修炼事件产生**加成值**（受段位倍率影响）
- 公式：`最终属性 = 基础值 + Σ(事件加成 × 段位倍率)`

## 修炼方式

### 关卡任务
每段设 2-4 个关卡，完成关卡获得属性奖励。每个关卡有**验证题**（服务端校验），答对才能标记完成。

### 段位考核
每段一套考核题（从 102 题题库中按类别均衡抽题）：
- 入门：6 题，答对 4 题通过
- 初段及以上：6 题，答对 ≥75% 通过
- 通过当前段位考核 → 自动突破到下一段

### 其他修炼事件
- 阅读推荐资源（+theory）
- 解锁新工具（+tools）
- 写感悟日记（+coding +theory）
- 社区发帖（+theory）
- 评论互动（+theory）

## 每段能力要求

### 入门
- 基本计算机操作、打字、使用浏览器
- 了解编程是什么

### 初段
- 变量与控制流、基本数据类型
- 会用开发工具和命令行
- 能运行代码、理解基本计算逻辑

### 二段
- 函数与模块化、文件操作
- 版本管理（Git）、包管理与依赖
- 能独立写工具解决小问题

### 三段
- 应用开发框架、数据持久化
- 网络与接口基础、基本 UI 构建
- 能做出完整可用的应用程序

### 四段
- 系统部署与运维、数据库设计
- 安全基础（认证/授权/加密）
- 端到端完成项目，从开发到运维

### 五段
- 自动化测试与 CI/CD
- 性能调优、中间件（缓存/队列）
- 独立维护线上系统

### 六段
- 系统架构设计、分布式基础
- 团队协作与 Code Review
- 设计系统、做技术决策

### 七段
- 底层原理（OS/编译/网络栈）
- 高并发与大规模系统
- 深入底层，突破常规限制

### 八段
- 架构与工具创新
- 开源贡献与技术布道
- 创造被认可的轮子

### 九段
- 范式创新、行业影响力
- 传道授业
- 定义了方向，推动了行业

## 设计说明

### 核心衡量标准不是「会什么」而是「能做什么」
看了多少教程、收藏了多少文章都不重要。有没有写出能用的东西、有没有让用户访问过、有没有独立解决过问题——**这才是真功夫。**

### 每段产出就是晋级条件
满足一条就算到了，不需要全满足。核心是**有人用、有产出。**

### 为什么删掉了数学前置？
实际上很多人做 Web 开发用到的高等数学很少，真正需要数学的时候自然就会去补。**六段以后**系统设计和性能优化才真正需要数学思维。

## 附录：趣味设定

### 异火 · 专精方向
- **异火·前端** — 精于 UI/交互
- **异火·后端** — 服务端高手
- **异火·数据** — SQL/大数据方向
- **异火·安全** — 安全攻防
- **异火·AI** — 机器学习和 AI 方向

### 副本 · 挑战任务
- **单人副本**：刷 LeetCode / 参加算法竞赛
- **组队副本**：参加 Hackathon / 做团队项目
- **世界 Boss**：修线上紧急 Bug / 性能优化挑战

### 突破感悟
每次提升一段，写一段「突破感悟」，记录下来当时学会了什么、踩了什么坑。
