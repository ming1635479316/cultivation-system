# 编程修炼体系 v2.0

> 九段制 · 以实际产出衡量境界

---

## 一段

**定位：** 刚摸到门，还在搞明白编程是怎么一回事

**能做什么：**
- 跟着教程把代码敲进去，跑起来
- 在编辑器里写个 Hello World
- 会改改变量值看看输出变化

**核心能力：**
- 认识变量、if else、for/while 循环
- 知道数组/列表是什么
- 能读懂简单的报错信息（知道哪一行出了问题）
- 会用 VS Code 写代码、运行

**典型产出：**
- 打印九九乘法表
- 猜数字小游戏
- 简单计算器

**这段最怕：** 遇到报错就慌、不知道怎么排查问题、只复制粘贴不理解代码意思

---

## 二段

**定位：** 能自己写点能用的东西了

**能做什么：**
- 独立写一个命令行小工具，解决一个实际问题
- 能把一个功能拆成几个函数
- 会使用别人写的库/模块

**核心能力：**
- 函数定义和调用、参数传递
- 字符串操作、数组/列表/字典的增删改查
- 文件读写
- 会用包管理器（pip/npm）装依赖跑起来
- git init、add、commit、push

**典型产出：**
- 批量重命名文件的小工具
- 简单的待办清单（命令行版）
- 从网页爬一段数据存成本地文件

**这段最怕：** 代码全在一个文件里、不会用函数、函数命名随便乱来

---

## 三段

**定位：** 能做出东西了，项目有模有样

**能做什么：**
- 独立完成一个带界面的小项目（网页/桌面/App）
- 会用框架，不再是光杆司令写代码
- 数据能存到数据库里

**核心能力：**
- 至少一个 Web 框架的基本使用（Flask/FastAPI/Django/Express 之一）
- SQL 增删改查、联表查询
- HTML + CSS 能写出结构还行的页面
- HTTP 基本概念：URL 是什么、GET 和 POST 啥区别、状态码 200/404/500 啥意思
- Git 分支、合并

**典型产出：**
- 个人博客（能写文章、能展示）
- 备忘录/记账本 Web 应用
- 带数据库的留言板

**这段最怕：** 只会照着教程走，离开了教程自己从头搭一个就不会了；遇到 Bug 不知道怎么排查

---

## 四段

**定位：** 能掌控整个技术栈，独立搞定完整项目

**能做什么：**
- 全链路独立开发：从设计到编码到部署上线
- 前后端都能写，数据库也能设计
- 项目结构清晰，不是一堆文件堆在一起

**核心能力：**
- 了解 RESTful API 设计，能写接口文档
- 用户认证（登录注册、JWT/Session）
- 前端框架（React/Vue）能做出交互式页面
- ORM 使用、数据库表结构设计
- Linux 基本操作（ssh、ls、cd、ps、grep）
- Nginx 反向代理基本配置
- Docker 基础（写个 Dockerfile 把项目打包）

**典型产出：**
- 完整上线的个人项目（有域名、能访问）
- 一个带用户系统的 Web 应用
- 能展示给别人用的东西

**这段最怕：** 项目只在自己电脑上跑过；部署遇到问题就放弃了；数据库不会设计，全塞一个表里

---

## 五段

**定位：** 能独挡一面，不用人带也能搞定事

**能做什么：**
- 代码质量有意识，不写屎山
- 出了问题能自己排查和修复
- 能看懂别人代码，能接手一个现有项目

**核心能力：**
- 单元测试（至少核心逻辑要有）
- 代码调试能力：加日志、断点调试、二分法定位问题
- SQL 优化：能看出慢查询，会加索引
- Redis 基础使用（缓存数据）
- CI/CD 基本流程（GitHub Actions / 类似工具）
- 能搭建基础的监控告警（错误日志收集、服务器监控）

**典型产出：**
- 参与/贡献一个开源小项目
- 旧项目的重构改进
- 独立上线并运营一个小项目（维护几个月）

**这段最怕：** 代码跑通就行不管质量；不自测就上线；遇到生产问题不知道怎么排查

---

## 六段

**定位：** 能设计、能带队、能决策

**能做什么：**
- 画得出系统架构图，说得清为什么这么拆
- 能带 2-3 个人干活，做 Code Review
- 能写技术方案，说得服别人

**核心能力：**
- 系统设计基础：服务拆分、API 网关、负载均衡
- 消息队列（Kafka / RabbitMQ 之一）基本使用
- 高可用：主从、集群、熔断、限流的概念
- 数据库进阶：读写分离、分库分表什么时候用
- Docker Compose / K8s 基础
- 安全常识：SQL 注入、XSS、CSRF、HTTPS
- 技术方案撰写与评审

**典型产出：**
- 设计并实现一个中型系统
- 带领小团队完成一个项目交付
- 写出一份能得到认可的技术方案

**这一段往往是很多人的瓶颈期，** 从写好代码到设计系统的跨越，需要大量实战积累。

---

## 七段

**定位：** 深入底层，能突破常规难题

**能做什么：**
- 能搞定高并发/大数据量场景
- JVM/OS 级别调优
- 能把一个复杂业务抽象成简洁的技术方案
- 能把业务问题转化为技术问题再解决

**核心能力：**
- 分布式系统理论：CAP、BASE、一致性哈希
- 性能调优：全链路压测、火焰图、GC 调优
- 中间件选型：能在 MySQL/Redis/ES/ClickHouse 里选对的
- 领域驱动设计
- 微服务架构设计与治理

这段要求就比较高了，**没踩过生产环境的坑很难到这一步。**

---

## 八段

**定位：** 能造轮子、有行业影响力

**能做什么：**
- 自己写的工具/库有人在用
- 在某个领域有专精深度
- 能输出方法论，指导别人成长

**核心能力：**
- 创造新工具/框架/语言的能力
- 开源社区贡献
- 深度掌握 3+ 语言并能讲出设计哲学
- 能把经验提炼成可复用的方法论

**典型产出：**
- 自己的开源项目有 100+ star
- 高质量的技术文章/分享
- 培养出六段以上的人

## 一些设计说明

### 为什么删掉了数学前置？
实际上很多人做 Web 开发用到的高等数学很少，真正需要数学的时候自然就会去补。把它写成「前置条件」会吓跑新人。到 **六段以后** 系统设计和性能优化才真正需要数学思维。

### 核心衡量标准不是「会什么」而是「能做什么」
你看了多少教程、收藏了多少文章都不重要。你有没有写过能用的东西、有没有让用户访问过、有没有独立解决过问题——**这才是真功夫。**

### 每段的「典型产出」就是你的晋级条件
满足了一条就算到了，不需要全满足。核心是**有人用、有产出。**

### 时间参考（因人而异）

| 段位 | 全职学习/工作时间参考 |
|:---:|:-------------------:|
| 一段 → 二段 | 1-3 个月 |
| 二段 → 三段 | 3-6 个月 |
| 三段 → 四段 | 6 个月 - 1 年 |
| 四段 → 五段 | 1 - 2 年 |
| 五段 → 六段 | 2 - 3 年 |
| 六段以上 | 5 年+ |

> 如果你不是全职，时间翻倍也很正常。别焦虑，慢慢来。

## 附录：趣味设定（可以加在网站里）

### 异火 · 专精方向
- **异火·前端** — 精于 UI/交互，HTML/CSS/JS 驾驭自如
- **异火·后端** — 服务端高手，性能优化精通
- **异火·数据** — SQL/大数据方向专精
- **异火·安全** — 安全攻防，渗透测试
- **异火·AI** — 机器学习和 AI 方向

### 副本 · 挑战任务
- **单人副本：** 刷 LeetCode / 参加算法竞赛
- **组队副本：** 参加 Hackathon / 做团队项目
- **世界 Boss：** 修线上紧急 Bug / 性能优化挑战

### 突破感悟
每次提升一段，写一段「突破感悟」，记录下来当时学会了什么、踩了什么坑。以后回看会很有意思。
